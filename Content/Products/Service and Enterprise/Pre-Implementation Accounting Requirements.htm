Pre-Implementation Accounting Requirements

Overview

In preparation for the day of Implementation below we’ve listed some requirements to consider and / or action before the day. As always, please call or email us with any questions.

Data integrity checklist

This is the perfect time for a spring clean. Spend time with your existing system and make sure you’re going to be happy with the data you're bringing across.

  • Remove duplicate customers & sites.
  • Archive or disable obsolete records. These may include customers, and / or site records and files.
  • Make sure all customer and site data you’d like imported to simPRO Enterprise exists in your accounting system.

Chart of Accounts

We’ll require your Chart of Accounts to ensure the Accounting Link is pushing the financial transactions to the appropriate accounts. On the day we can discuss this further. It will ultimately be your decision, ideally in consultation with your accountant.

Non-accounting data

Data extraction from other systems or data modification is the responsibility of the client. Once all required data is extracted the Implementation consultant will check it is in the appropriate format. Another option is to engage an external consultant such as: convertworx.com.au

Linking your accounting package to simPRO Enterprise

On day one of Implementation, the Accounting Link will be installed and configured on whichever PC has direct access to the executable file and the data files of your accounting software. Ensure you have administrator access to the PC and / or the server hosting the simPRO Accounting link.

We strongly recommend the person responsible for your accounting is present on the first day.

If you’re unsure about your accounting software’s compatibility with simPRO Enterprise, contact us as soon as possible.

Cloud accounting software

All we require here is the Administrator login details. MYOB Live, Xero and Quickbooks Online are completely compatible with the simPRO Accounting Link.

MYOB AccountRight Live

AccountRight Live gives you the option to store your data file on either your desktop or to upload the data file to AccountRight Live. In order for the simPRO Accounting link to work, you must upload the file to AccountRight Live.

The AccountRight Live link will not import employee cards.

The simPRO Accounting Link cannot transfer data when MYOB is in ‘Offline (check out)’ mode.

Customer and Supplier card files match the NAME field between MYOB and Enterprise. Therefore, if you manually create a customer in Enterprise named ‘Peter Lee’, as long as in MYOB it is called the exact same, accounting link will treat them as the same customer. Therefore, if there’s an invoice been created in Enterprise, it will be sent over against ‘Peter Lee’ in MYOB. If you change the name of the card file in simPRO, you can have change the name of that card file in MYOB as well. Otherwise, a new card file with new name will be created in MYOB.

Xero

Xero is true cloud software, so you don't have to install anything.

We cannot import employees or contact cards from Xero, only customer and site cards.

If you’ve recently migrated to Xero from another accounting package, the customers will be recorded as Contacts only until a transaction is recorded against them. Please contact our office and speak to Trainer on Call to get in touch.

QuickBooks Online

During the initial import, customers and suppliers can be imported from QuickBooks Online to simPRO.

 Desktop accounting software

To integrate with desktop accounting packages we require:

Net framework 4.0 or above. This should be installed on the computer where your accounting software resides. If this is not currently the case please discuss with your IT professional. The simPRO trainer cannot perform this action as it may impact other systems.

MYOB

The Mac version of MYOB is not supported by the simPRO Accounting Link.

To interface with MYOB for the first time you’ll need to register your company file for an ODBC transaction.

To register:

Visit https://my.myob.com.au/Pages/ODBCActivation.aspx and follow the prompts

Or call MYOB support on 1300 555 123 and let the consultant know that you wish to register the file for ODBC. Quote Developer Code: 7083501

Sage 50

You will need to obtain the Activation key from Sage and install it on every PC that connects to the Sage data.

To obtain the Activation key, log into Sage and go to Sage > Tools > Activation > Enable 3rd party integration. Follow the instructions provided.

Write access to the drive on which you would like to install the Sage link. If an external company to manages your system, you may have limited access.

You'll also need to let us know what Sage add-ons you are using. To find the add-ons, go to Sage > Help > About.

 What if my accounting package is not listed?

simPRO Software will not extract data from legacy systems or non-compliant accounting packages, we can however recommend suitable partners to help prepare data from such systems into simPRO Enterprise.

 Important note regarding on-site servers

We do not support on-site servers.

If you’re currently running an on-site server or terminal server for your accounting package, please let us know. We’ll need to discuss what may be required to tidy up your database, for example, remove inactivate customer and site card files no longer in use, or merge duplicated card files (Applicable for MYOB, Sage and QuickBooks only).

If you’re running Mac OS X please call to check compatibility of the accounting link.
