Asset Service Levels

Overview

A service level details the service and testing frequency for a customer asset. You can assign a service level to an asset type, create failure points (often called 'testing points'), and optionally define recommendations for rectifying any asset failures.

When you add a new asset to a site, all service levels set up under the asset type will be added to the asset. You can add or remove service levels per site to suit the service levels required by your contracts.

Create a service level

A service level must be set up before you can assign it to an asset type.

To create a service level:

  1. Go to System > Setup > System Setup > Customer Assets > Service Levels.
  2. Click Create Service Level.
  3. Enter a Service Level Name.
  4. Enter the desired frequency from the Years, Months, and Days fields.
  5. Click Create.
 Edit a service level

To edit a service level:

  1. Go to System > Setup > System Setup > Customer Assets > Service Levels.
  2. Click the service level you wish to edit.
  3. Make changes as desired.
  4. Click Save.
 Archive a service level

If a service level has history, it can be archived rather than deleted. Users without the relevant security group settings will not be able to perform this function.

To archive a service level:

  1. Go to System > Setup > System Setup > Customer Assets > Service Levels.
  2. On the service level you wish to archive, click the red delete icon.
  3. Select removal options as desired. These options will bulk-remove service levels from assets and asset types.
  4. Click OK.

If a service level has been archived but not removed from asset types, it will display as (removed).

While archiving a service level is reversible, removing service levels from asset types and assets cannot be undone. If you use this function to remove a service level from many assets or asset types at once, then want to re-assign it, you will have to do it manually for each asset or asset type, as well as re-link set price pre-builds.

 View archived service levels

To view archived service levels:

  1. Go to System > Setup > System Setup > Customer Assets > Service Levels.
  2. To the right of Service Level Name, select Show Archived?.

Archived service levels display a green add icon, while non-archived service levels display a red delete icon.

 Un-archive a service level

Users without the relevant security group settings will not be able to perform this function.

To un-archive a service level:

  1. Go to System > Setup > System Setup > Customer Assets > Service Levels.
  2. Select Show Archived?.
  3. On the service level you wish to un-archive, click the green add icon.
  4. Click OK.

​Repeat the process to restore multiple service levels.

Un-archiving a service level will not re-assign it to any asset types or assets it was previously assigned to.

 Bulk-remove service levels from assets

If a service level ever becomes defunct, there are a number of ways to remove a service level from many assets at once.

We recommend carefully restricting security group access to archiving assets.

Removing service levels from asset types and assets cannot be undone. If you remove a service level from many assets or asset types at once, then want to re-assign it, you will have to do it manually for each asset or asset type, as well as re-link set price pre-builds. As this is not reversible, we recommend taking a snapshot of your database before removing service levels from many assets.

Remove a service level from all assets not associated with contracts

To remove a service level from all assets not associated with contracts:

  1. Go to System > Setup > System Setup > Customer Assets > Service Levels.
  2. On the service level you wish to remove, click the red delete icon.
  3. Select Remove Service Level from all Assets not associated with Contracts.
  4. Click OK.
 Remove a service level from all assets

To remove a service level from all assets in your system:

  1. Go to System > Setup > System Setup > Customer Assets > Service Levels.
  2. On the service level you wish to remove, click the red delete icon.
  3. Select Remove Service Level from all Assets (this will also remove it from all assets not associated with contracts).
  4. Click OK.
 Remove a service level from all asset types

To remove a service level from all asset types:

  1. Go to System > Setup > System Setup > Customer Assets > Service Levels.
  2. On the service level you wish to remove, click the red delete icon.
  3. Select Remove Service Level from all Asset Types.
  4. Click OK.
 Remove a service level from all assets of an asset type

To remove a service level from all assets of a certain asset type:

  1. Go to System > Setup > System Setup > Customer Assets > Asset Types.
  2. Open the asset type you wish to remove a service level from.
  3. Click the Service Levels sub-tab.
  4. Click Options > Delete Service Level.
  5. Select Remove from all Assets of this type not associated with Contracts or Remove from all Assets of this type, as applicable.
  6. Click OK.
 Remove a service level from a site's assets

To remove a service level from assets listed under a specific site:

  1. Go to People > Sites.
  2. Click the relevant site.
  3. Click the Customer Assets tab.
  4. Select the assets you wish to remove a service level from, or click Select All from the bottom of the page.
  5. At the bottom of the page, click Remove Service Levels.
  6. Select the service levels you wish to remove.
  7. If desired, select Exclude Assets selected associated with a contract.
  8. Click OK.

To include contracts in the table view of a site's Customer Assets tab, go to Utilities > Customer Assets, click an asset type, click Modify Table View, select Contract and click Finish. Users without the relevant security group settings will not be able to perform this function.

 Assign a service level to an asset type

After setting up service levels, you can assign them to asset types.

To assign an existing service level to an asset type:

  1. Go to System > Setup > System Setup > Customer Assets > Asset Builder.
  2. Click the asset type you wish to assign a service level to.
  3. Click the Service Levels tab.
  4. Select a service level from the Service Level Name drop-down list and click Add.
  5. Repeat as desired.
  6. Click Finish.
 Pre-build charge rates

As you add a service level to an asset type, you can select a default charge rate to bill your customers. This charge rate will appear as a billable item in any jobs raised for that asset type and service level through the Maintenance Planner, and will appear on itemised invoices.

Create charge rates by creating set price pre-builds. You can structure pre-builds to charge each customer the same amount for the same service level.

If the price differs by customer, set a generic value of $0 and use customer contracts to manage customer-specific pricing, under People > Customers > a customer > Rates > Contracts. These prices can also be modified manually on a job, which is best when you bill customers per asset, per site, per visit.

After setting up pre-builds, you can link them to the service level of an asset type by selecting them in the Charge Rate field when assigning a service level.

Set price pre-builds can also be applied to Failure Points and Recommendations, which will be used for rectification quotes.
