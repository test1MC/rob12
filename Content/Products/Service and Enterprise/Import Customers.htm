Import Customers

Overview

In simPRO, you can manually add customers to the system through People > Create New Customer, or save time and reduce data entry errors by importing a customer list.

You may wish to test the import process with one or two customers before importing an entire list.

You may choose to include a Notes column containing details about a customer to upload to the Notes text box in Profile > Settings in the customer card file. Alternatively, you can import customer notes to be viewed in the Notes tab of the customer card file.

If customers were created during an initial accounting package import, adjust your If Customer Already Exists import settings accordingly so as to not create duplicate customer card files.

If a customer has a numeric-only custom field in simPRO but there is non-numeric data entered in the field in the customer list, you will receive an error warning and the import will fail. Numbers must be formatted as specified in System > Setup > Defaults > System > Number Thousands Separator / Negative Number Format to import successfully.

Import a customer list

After preparing your customer list, work through the following steps to import your list into simPRO.

Adjust import settings

To adjust customer import settings, go to Utilities > Import > Customers > Customers.

The options available include:

First Record Is

Select Header Record if your list's first row contains column titles, or select Data Record if your list's first row contains specific data.

If Customer Already Exists

In case simPRO finds duplicated customers (customers with the same name), select whether you want to Do Nothing or Update Customer Details for the existing site in simPRO, or alternatively, Always Create to upload all customers in the file to simPRO.

Create New Site

Select Yes if you want to create new site card files to be associated with the applicable customers. Select No if you do not wish to create new sites.

If Custom List Value Not Found

If any of the entries are a drop-down list in simPRO (for instance, in Custom Fields or the State drop-down list in the Details tab of the customer card file), and your file contains any additional values (or 'custom list values') not available in the relevant list, select Create New List Option to add the additional value to the drop-down list in your customer card files. Select Reject and Warn to ignore the field and receive a warning when the customer list has been uploaded that the field was not created.

Customer Status

Select whether the upload file is a Customer or Lead list.

 Select an import file

To select an import file:

  1. Go to Utilities > Import > Customers > Customers.
  2. Adjust customer import settings, then click Choose File / Browse.
  3. Locate your customer import file and click Open.
  4. Click Next.
 Match import columns

After selecting a customer import file, manually match or review your list columns with the fields available in simPRO. You may wish to have both simPRO and your list open at the same time.

To match import columns:

  1. Select your import file as above.
  2. On the Customer Import Columns page, the column headings from your imported list will appear in brackets, such as (Company Name). Use the drop-down list under each heading to select the matching field in simPRO.
  3. If you wish to leave a column out of the import entirely, select Ignore.
  4. Click Import Customers.
 Check import success

After uploading an import file, the Import Customer Summary page will display a breakdown of your import, including Customers Added / Updated / Ignored and Sites Created.

Click Customers imported successfully to download a CSV file of imported customers for your records, with each customer line containing their new simPRO-assigned Customer ID.

After a successful import, it is recommended you manually check customer card files to ensure customer data has imported correctly. To do this, go to People > Customers, click an imported customer, and check Profile > Details for errors. Manually amend if required.

 Amend import errors

If your customer import file does not contain a First & Last Name for individual customers or a Company Name for companies, the import will fail and you will receive an error message.

To amend the error, click OK, then enter details in the required fields in your import list and restart the import process.

If your file contains other errors, an error message will display on the Import Customer Summary page, and the table will display the number of errors found.

Click click here to download a CSV file of import errors. The Error Text column will explain why entries failed. Amend your customer import list accordingly and restart the import process.
